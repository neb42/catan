  <name>Task 1: Create Port.tsx Component</name>
  <files>apps/web/src/game/Board/Port.tsx</files>
  <action>
Create a new Port component that renders port visuals on the board.

**Port data format (server-generated):**
- Port schema: `{ position: string, type: '3:1' | '2:1-wood' | '2:1-wheat' | '2:1-sheep' | '2:1-brick' | '2:1-ore' }`
- Position string format: `"q,r:directionIndex"` where directionIndex is 0-5
  - Example: `"0,1:3"` means hex at axial coord (0,1), direction index 3
  - Direction indices 0-5 map to the 6 hexagon edges (clockwise from east)
- Generated in apps/api/src/game/BoardGenerator.ts lines 115-133

**Component implementation:**
1. Accept port prop: `{ type: string, position: string }`
2. Parse `position` string to extract hex coord and edge direction:
   ```typescript
   const [coords, directionStr] = position.split(':');
   const [q, r] = coords.split(',').map(Number);
   const directionIndex = parseInt(directionStr);
   ```
3. Convert axial (q,r) to pixel position using geometry.ts utilities
4. Calculate port icon placement on the coastal edge based on directionIndex (0-5):
   - Direction 0: East edge
   - Direction 1: Southeast edge
   - Direction 2: Southwest edge
   - Direction 3: West edge
   - Direction 4: Northwest edge
   - Direction 5: Northeast edge
5. Render port icon/indicator based on type:
   - Generic "3:1" ports: Use a `?` symbol or ship icon with "3:1" label
   - Resource-specific "2:1" ports: Parse resource from type string (e.g., "2:1-wood") and display resource icon + "2:1" label
6. Style with SVG elements (rect/circle for background, text for labels)
7. Position port indicator outside the hex on the coastal edge
8. Use consistent sizing (~30x30 pixel box for visibility)

**Port type parsing:**
- `type === "3:1"` → generic port (any resource)
- `type === "2:1-wood"` → wood-specific 2:1 port (extract "wood")
- Similar for wheat, sheep, brick, ore

**Reference existing components:**
- apps/web/src/game/Board/Settlement.tsx for cubic-to-pixel coordinate conversion pattern
- apps/web/src/game/Board/geometry.ts for coordinate utilities (axialToPixel)

**Why not hand-roll:** Use existing geometry utilities from geometry.ts to maintain coordinate system consistency. Do not create new conversion functions.
